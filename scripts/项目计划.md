# 博客图片迁移项目计划书

## 项目目标

将博客文章中所有外部图片链接（CSDN、smain 图床等）批量下载到本地，并自动替换 Markdown 文件中的图片链接为本地路径。

## 需求分析

### 当前状况
- **文章数量**：10 篇
- **图片引用**：约 159 处
- **独立图片**：约 164 张
- **图片来源**：
  - `api.smain.cn` 图床（主要）
  - `i-blog.csdnimg.cn` CSDN 图床

### 目标结构
```
source/
├── _posts/                          # 博客文章目录
│   ├── LeetCode1863-xxx.md
│   ├── Linux-xxx.md
│   └── ...
└── images/                          # 图片存储目录（新建）
    ├── LeetCode1863-找出所有子集的异或和再求和【简单】/
    │   ├── 1.png
    │   ├── 2.png
    │   └── ...
    ├── Linux（Centos7）安装 docker、mysql 踩坑总结/
    │   ├── 1.png
    │   ├── 2.png
    │   └── ...
    └── ...
```

## 实现方案

### 步骤一：创建目录结构
- 在 `source/` 下创建 `images/` 目录
- 每篇文章对应一个子目录

### 步骤二：编写 Python 脚本

**脚本功能模块**：

1. **扫描模块** (`scan_posts.py`)
   - 遍历 `source/_posts/` 目录下所有 Markdown 文件
   - 使用正则表达式提取所有图片链接 `![alt](url)`
   - 建立文章与图片 URL 的映射关系

2. **下载模块** (`download_images.py`)
   - 根据文章名创建对应子目录
   - 按图片出现顺序编号（1.png, 2.png, ...）
   - 使用 `requests` 库下载图片
   - 添加重试机制和错误处理

3. **替换模块** (`replace_links.py`)
   - 读取 Markdown 文件内容
   - 将外部 URL 替换为本地路径 `/source/images/文章名/序号.png`
   - 保存修改后的文件

4. **主程序** (`main.py`)
   - 整合以上模块
   - 提供进度显示
   - 输出执行日志

### 步骤三：执行迁移

```bash
cd scripts
python main.py
```

### 步骤四：验证结果

1. 检查 `source/images/` 目录结构
2. 检查 Markdown 文件中的图片链接是否已替换
3. 运行 `npm run server` 预览博客
4. 验证所有图片是否正常显示

## 技术细节

### 正则表达式
```python
# 匹配 Markdown 图片语法：![alt](url)
IMAGE_PATTERN = r'!\[([^\]]*)\]\(([^)]+)\)'
```

### 目录命名
- 使用文章文件名（不含 `.md` 后缀）作为图片目录名
- 例如：`LeetCode1863-找出所有子集的异或和再求和【简单】/`

### 图片命名
- 按文章中出现的顺序编号：`1.png`, `2.png`, `3.png`...
- 保持原图片格式（.png/.jpg/.gif 等）

### 错误处理
- 网络请求超时重试（3 次）
- 下载失败的图片记录到日志
- 跳过已存在的图片（避免重复下载）

## 风险与应对

| 风险 | 应对措施 |
|------|----------|
| 网络请求失败 | 重试机制 + 错误日志 |
| 图片链接无法访问 | 跳过并记录，手动处理 |
| 目录名包含特殊字符 | 使用安全字符替换 |
| 替换后路径错误 | 执行前备份原文件 |
