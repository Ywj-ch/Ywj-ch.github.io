## 三、优化方案

### 3.1 目录结构优化

#### 方案 A：按年份分类（推荐）⭐

```
source/images/
├── 2025/
│   ├── leetcode-1863/           # LeetCode1863-异或和
│   ├── leetcode-368/            # LeetCode368-最大整除子集
│   ├── leetcode-416/            # LeetCode416-分割等和子集
│   ├── linux-docker-mysql/      # Linux 安装 docker、mysql
│   ├── springboot-admin/        # 基于若依框架的 SpringBoot 管理系统
│   └── apple-problem/           # 分苹果问题
├── 2024/
│   ├── hexo-blog-setup/         # Hexo 博客搭建
│   ├── microservice/            # 微服务入门
│   └── python-fiddler/          # Python 与 Fiddler 图书馆预约
└── common/                      # 公共资源
    ├── default-cover.jpg
    └── 404.jpg
```

**优点**：
- ✅ 目录名简短，全英文
- ✅ 按年份分类，易于查找
- ✅ Git 兼容性好

**缺点**：
- ❌ 需要更新 10 篇文章的图片路径

---

#### 方案 B：简化当前目录名

```
source/images/
├── leetcode-1863-xor/
├── leetcode-368-subset/
├── leetcode-416-partition/
├── linux-docker-mysql/
├── springboot-ruoYi/
├── apple-problem-binary/
├── hexo-butterfly-setup/
├── microservice-springcloud/
└── python-fiddler-library/
```

**优点**：
- ✅ 保持现有文章目录结构
- ✅ 只需重命名目录

**缺点**：
- ❌ 扁平结构，无分类
