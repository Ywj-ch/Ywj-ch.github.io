为什么写博客

其实最开始写博客的主要原因还是因为方便自己。学完一门新技术之后总是兴奋的，想要做点什么，而对于我们程序员来说，记录开发过程中的经验可以帮助你回顾你所掌握的知识。我们这一行业技术更新迭代太快了，需要我们持续的学习。写博客就相当于是在记笔记吧。

关于我的经历

我最开始系统性接触的语言是 Java，和大多数学 Java 的人一样，从后端技术开始学起。差不多学了半年左右的时间吧，那时候是大三下刚开学（25 年春），我准备投一投实习。或许是自身能力的不足吧，没有能够取得自己想要的结果。

后面一段时间就随大流跑去考研了。一方面是缓解就业压力吧，另一方面也确实是我感觉自己还有很多想要学的东西，还有很多想要做的东西，但是受限于自身能力而不能及。

中间有半年左右的时间我都没有怎么更新博客了。不得不承认，写博客真的是一件比较耗时的事情——如果你想要清晰的表达自己的思路的话。不过最近我发现很多本地运行的 Agent 可以帮你解决一些问题了，你可以跟他聊一聊你的想法、你的心得体会，然后他就能帮你写出还不错的框架了。我觉得这对于我写算法总结来说还是不错的，不过我还没有试过用来写更长的文章。

扯远了。

考研与读书

初试之后，我就立马投入到了毕业设计的开发当中。当然我是喜悦的，因为终于有时间可以做做自己的项目了。不过或许是还要继续准备复试的压力吧，我没有给自己选一个特别有难度的题目，而是折中了一些。虽然最后答辩的时候老师都觉得我做的还不错，但是我自己清楚这不算什么。

研究生考试这段时间正好我们的专业课要考 C++，所以又趁着这段时间好好学了一下 C++。确实 C++ 的内容很多，除了教材之外我也看了一些相关的书——Effective 三部、深度探索 C++ 对象模型。哦还有推荐 YouTube 上一个叫 Cherno 的博主，他讲的 C++ 课程确实很不错，内容挺有深度的，不过是英文教学，虽然有字幕但是整体节奏还是偏快的，需要有一定的基础。

不得不说在备考的这段时间还是看了一些书，而且让我越发感到阅读的重要性。程序员只需要会编程、会技术、不用花太多时间看书——这种说法我觉得还是有点问题的。可能由于现在我还是学生的原因吧，所以还有时间可以静下来看书，等到以后工作之后这种时间会变得越来越奢侈吧？或许这也是我选择继续呆在学校的原因。

毕设与 AI

再聊一聊毕设。我主要是做的一些前端和CV相关的东西，不得不说 AI 在前端的能力真的是令人震惊的——难怪网上说冲击如此之大。我是没什么前端经验的，所以还是用着之前后端开发的思维逻辑来指导 AI 开发，效果竟然意外地不错，过程中也积累了很多的经验。

不过也开始越来越担心起来。AI 这个东西确实可以将你的能力放大数倍，你可以做出超出你能力范围之外很多的事情，不过这会给你一种虚幻的感觉，一种似懂非懂的感觉。我觉得这个问题真的挺难解决的，因为当效率被摆在第一位的时候，你很难跳出这一种舒适圈，但这其实是对初级开发者不太有利的。

现在 & 未来

说实话我是有点闲不下来的。一方面可能是因为习惯了之前的作息规律，一方面又可能还是有些焦虑吧。研究生的方向选的是人工智能相关研究方向，这没办法，你得跟着趋势走——就算以后出来不一定能够从事这方面的研究，但是目前来说我觉得没有更好的选择了。

看书，还是看书。机器学习、深度学习，学的越多越感觉自己学得不够。内容太多了，分枝太多了，不仅得学理论，还要学应用。其实还是会有点担心就业的问题吧，所以感觉一直有点迷茫，想要找出一条最高效的路。但是说实话这真的挺凭运气的，没有谁能够预料到几年甚至几个月之后会发生的事情。走对方向的人自然会一帆风顺，但稍微有点偏航其实感觉也没有那么严重。

最近在知乎上刷到李沐写的一篇博客，我觉得对我来说挺有感触的，值得好好读一读：

要有一个目标。 短也好，长也好，认真定下的也好，别人那里捡的也好。就跟随机梯度下降需要有个目标函数一样。

目标要大。 不管是人生目标还是目标函数，你最好不要知道最后可以走到哪里。

坚持走。 每一次你找一个大概还行的方向，然后迈一步。但最重要的是你得一直走下去，能多走几步就多走几步。

四处看看。 你需要四处走走，看过很多地方，做些错误的决定，这样你可以在前期迈过一些不是很好的舒适区。

痛苦的卷。 改变带来痛苦。但没有改变就没有进步。需要时刻跟自己作对。

可以躺平。 步子太小走不动，步子太长容易过早消耗掉了激情。周期性的调大调小步长效果挺好。所以你可以时不时休息休息。

关注当前，每次抬头瞄一眼世界，快速做个决定，然后迈一小步。小步快跑。只要你有目标，不要停，就能到达。

(《用随机梯度下降来优化人生》)

雄关漫道真如铁，而今迈步从头越。

还望诸君共勉之！
